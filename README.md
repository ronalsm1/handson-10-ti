## Catatan Pembelajaran
1. SOLID principles dipopulerkan oleh Robert C. Martin yang dikenal sebagai Uncle Bob, seorang software engineer dan penulis buku terkenal, pada awal tahun 2000-an.
2. SOLID adalah akronim dari lima prinsip desain dalam OOP yang bertujuan untuk membuat software lebih mudah dipahami, fleksibel, dan maintainable.
3. Single Responsibility Principle berarti sebuah class seharusnya hanya memiliki satu tanggung jawab atau satu alasan untuk berubah.
4. Open/Closed Principle berarti class seharusnya terbuka untuk diperluas (extension), tapi tertutup untuk dimodifikasi (modification).
5. Liskov Substitution Principle berarti objek dari parent class harus bisa digantikan dengan objek dari child class tanpa mengubah behavior yang benar dari program.
